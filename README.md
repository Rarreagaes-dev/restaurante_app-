# Sistema de Gestión de Restaurante

## Estudiante

Ruth Noemí Arreaga Espinoza

## Descripción

Este proyecto corresponde a la actividad de la Semana 5 de la asignatura Programación Orientada a Objetos. El sistema permite registrar productos y clientes de un restaurante mediante clases, objetos y listas, aplicando una estructura modular en Python.

## Estructura del proyecto

```
restaurante_app/
├── modelos/
│   ├── __init__.py
│   ├── producto.py
│   └── cliente.py
├── servicios/
│   ├── __init__.py
│   └── restaurante.py
└── main.py
```

## Tipos de datos utilizados

- **str:** nombre, categoría y correo.
- **int:** edad.
- **float:** precio.
- **bool:** disponibilidad del producto y cliente frecuente.
- **list:** almacenamiento de productos y clientes.

## Reflexión


Durante el desarrollo de esta actividad comprendí la importancia de utilizar identificadores descriptivos y tipos de datos adecuados para que el código sea más claro y fácil de entender. También aprendí que organizar un proyecto de forma modular permite mantener cada clase con una responsabilidad específica, lo que facilita su lectura, mantenimiento y reutilización. Considero que aplicar estos principios desde el inicio contribuye a desarrollar programas mejor estructurados y fortalece mis habilidades en Programación Orientada a Objetos para proyectos de mayor complejidad en el futuro.
